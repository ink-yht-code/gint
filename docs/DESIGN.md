# gint-gen 代码生成器设计文档

## 概述

gint-gen 是一个基于 DDD（领域驱动设计）+ Clean Architecture 的微服务代码生成器，支持 HTTP + gRPC 双协议，帮助开发者快速搭建标准化微服务架构。

---

## 核心设计决策

| 决策项 | 选择 |
|--------|------|
| 目录结构 | 一服务一目录 `<svc>/internal/...` |
| 配置 | 每服务独立 `configs/<svc>.yaml`，YAML 格式 |
| 日志 | zap，框架自动初始化，业务用 `log.Ctx(ctx).Info(...)` |
| DI | 手写 wiring，生成器生成骨架 |
| HTTP | Gin + gint，可选启用 |
| gRPC | protobuf，可选启用，同进程 |
| 事务 | service 控制，框架提供 TxManager（ctx 注入 DB） |
| 错误码 | 每服务独立 code 段，BizError 统一类型 |
| gRPC 错误 | status + details 带 BizCode |
| 领域模型 | 充血模型，规则在 domain 方法内 |
| 事件 | 预留 event，直接生成 outbox 表 + repo |
| Outbox payload | JSON |
| ServiceID 分配 | 中心化 Registry 服务 |
| API 定义文件 | `.gint` 扩展名 |

## 目录结构

```
user/                           # 服务根目录
  user.gint                     # API 定义文件（用户编辑）
  user.proto                    # gRPC 定义文件（可选）
  cmd/
    main.go                     # 入口（只调用 wiring.BuildApp + app.Run）
  configs/
    user.yaml                   # 服务配置
  internal/
    config/
      config.go                 # Config struct + Load + Validate
    domain/
      errs/
        codes.go                # ServiceID + BizCode 常量
        error.go                # BizError 类型
      entity/
        <agg>.go                # 聚合根：行为方法（充血模型）
      port/
        <agg>_repo.go           # repository 接口
        outbox_repo.go          # outbox 接口
      event/
        events.go               # 领域事件类型
    repository/
      dao/
        db.go                   # gorm helper
        <agg>_dao.go            # gorm 实现
      cache/
        <agg>_cache.go          # redis 实现
      outbox/
        model.go                # outbox gorm model
        repo.go                 # outbox repo 实现
    types/
      types.go                  # 请求/响应结构（生成，不要改）
    server/
      <usecase>.go              # 业务逻辑（用户编辑）
    web/
      routes.go                 # 路由注册（生成，不要改）
      <usecase>_handler.go      # HTTP handler（生成，不要改）
      http.go                   # HTTP server 启动
      rpc.go                    # gRPC server 启动（可选）
      middleware.go             # 中间件（用户编辑）
    wiring/
      wiring.go                 # NewDB/NewRedis/NewRepo/NewServer...
      app.go                    # BuildApp -> App
```

---

## 调用链路

```
web (handler) → server (业务逻辑) → repository (dao/cache) → DB/Redis
                   ↓
              domain (聚合根、实体)
```

**依赖方向**：
- `web` 只能依赖 `server` + `types`
- `server` 只能依赖 `domain` + `port`（接口）
- `repository` 实现 `port`，不依赖上层
- `domain` 不依赖任何基础设施

## 文件分类

### 生成文件（会被覆盖，不要手动改）

| 文件 | 说明 |
|------|------|
| `internal/types/types.go` | 请求/响应结构体 |
| `internal/web/routes.go` | 路由注册 |
| `internal/web/*_handler.go` | handler 骨架 |

每次执行 `gen api` 都会重新生成，手动修改会丢失。

文件头部会加注释：
```go
// Code generated by gint-gen. DO NOT EDIT.
// source: user.gint
```

### 用户文件（不会被覆盖，可以改）

| 文件 | 说明 |
|------|------|
| `<svc>.gint` | API 定义（源） |
| `internal/server/*.go` | 业务逻辑 |
| `internal/domain/**/*.go` | 领域模型 |
| `internal/repository/**/*.go` | 数据访问 |
| `internal/wiring/*.go` | 依赖组装 |
| `internal/config/*.go` | 配置 |
| `configs/<svc>.yaml` | 配置文件 |
| `cmd/main.go` | 入口 |

## API 定义文件格式（.gint）

```go
syntax = "v1"

type CreateUserReq {
    Name  string `json:"name" validate:"required"`
    Email string `json:"email" validate:"required,email"`
}

type CreateUserResp {
    Id    int64  `json:"id"`
    Name  string `json:"name"`
}

@server(
    prefix: /api/v1
)
service user {
    @handler CreateUser
    POST /users (CreateUserReq) returns (CreateUserResp)
    
    @handler GetUser
    GET /users/:id returns (GetUserResp)
    
    @handler ListUsers
    GET /users (ListUsersReq) returns (ListUsersResp)
}
```

## 错误码体系

### 编码规则

- 每个服务分配一个 `ServiceID`（3 位，从 101 开始）
- 业务码：`Code = ServiceID * 10000 + BizCode`

例如 user 服务（ServiceID=101）：
- 参数错误：`1010001`
- 未授权：`1010002`
- 无权限：`1010003`
- 内部错误：`1019999`

### BizError 类型

```go
type BizError struct {
    Code int
    Msg  string
    Cause error
}
```

### HTTP 错误映射

```go
// BizError -> gint.Result
func MapError(err error) (int, gint.Result) {
    var biz *BizError
    if errors.As(err, &biz) {
        switch biz.Code % 10000 {
        case 1: // InvalidParam
            return http.StatusBadRequest, gint.InvalidParam()
        case 2: // Unauthorized
            return http.StatusUnauthorized, gint.Unauthorized()
        case 3: // Forbidden
            return http.StatusForbidden, gint.Forbidden()
        default:
            return http.StatusInternalServerError, gint.InternalError()
        }
    }
    return http.StatusInternalServerError, gint.InternalError()
}
```

### gRPC 错误映射

```go
// BizError -> status + details
func MapError(err error) error {
    var biz *BizError
    if errors.As(err, &biz) {
        code := mapBizCodeToGrpcCode(biz.Code)
        st := status.New(code, biz.Msg)
        detail := &common.ErrorDetail{
            BizCode:   int32(biz.Code),
            BizMsg:    biz.Msg,
            RequestId: GetRequestIdFromContext(),
        }
        st, _ = st.WithDetails(detail)
        return st.Err()
    }
    return status.Error(codes.Internal, "internal error")
}
```

## 事务管理

### TxManager 接口

```go
type TxManager interface {
    Do(ctx context.Context, fn func(ctx context.Context) error) error
}
```

### 使用方式

```go
func (s *UserService) CreateUser(ctx context.Context, req *CreateUserReq) error {
    return s.tx.Do(ctx, func(ctx context.Context) error {
        // 在 ctx 中的 DB 已经是事务 DB
        user := domain.NewUser(req.Name, req.Email)
        if err := s.repo.Save(ctx, user); err != nil {
            return err
        }
        // 写入 outbox
        event := event.UserCreated{UserId: user.Id}
        return s.outbox.Save(ctx, event)
    })
}
```

### DB 获取方式

```go
// dao 层通过 ctx 获取 DB
func (d *UserDAO) Save(ctx context.Context, user *entity.User) error {
    db := dbx.FromContext(ctx, d.db) // 如果 ctx 有 txDB 用 txDB，否则用 baseDB
    return db.Create(user).Error
}
```

## 事件与 Outbox

### Outbox 表结构

```sql
CREATE TABLE outbox (
    id VARCHAR(64) PRIMARY KEY,
    service VARCHAR(32) NOT NULL,
    event_name VARCHAR(64) NOT NULL,
    payload JSON NOT NULL,
    status VARCHAR(16) NOT NULL DEFAULT 'pending',
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    next_retry_at TIMESTAMP,
    retry_count INT DEFAULT 0,
    request_id VARCHAR(64),
    INDEX idx_status_next_retry (status, next_retry_at)
);
```

### 领域事件

```go
type UserCreated struct {
    UserId    int64
    Name      string
    Email     string
    CreatedAt time.Time
}

func (e *UserCreated) Name() string {
    return "user.created"
}

func (e *UserCreated) OccurredAt() time.Time {
    return e.CreatedAt
}
```

## ServiceID Registry 服务

### API

- `POST /v1/services:allocate`
  - 入参：`{ "service_name": "user" }`
  - 出参：`{ "service_id": 101, "service_name": "user" }`
  - 幂等：同名重复请求返回相同 ID

### 存储

- SQLite/Postgres/MySQL
- 表：`service_name (unique), service_id (unique), created_at`

### 安全

- 简单 token 验证

## 生成器命令

```bash
# 创建新服务
gen new service user --transport http,rpc

# 从 .gint 文件生成 HTTP 代码
gen api user

# 从 .proto 文件生成 gRPC 代码
gen rpc user

# 从 SQL 生成 repository
gen repo user --ddl schema.sql
```

## 工作流程

```
第一次：
  gen new service user        → 生成服务骨架（所有文件）
  编辑 user.gint              → 定义接口
  gen api user                → 生成 types/handler/routes
  在 server/ 写业务逻辑        → 用户自己维护

后续：
  修改 user.gint              → 新增/修改接口
  gen api user                → 重新生成 types/handler/routes（覆盖）
  在 server/ 写业务逻辑        → 用户自己维护
```

## 分层约束（禁止跨包调用）

### 规则

- `web` 禁止 import `repository/**`
- `server` 禁止 import `gorm.io/*`、`github.com/redis/*`
- `domain` 禁止 import 任何基础设施库

## 框架运行时（gintx）

提供：
- `log`：全局 logger + ctx logger（自动注入 request_id）
- `tx`：TxManager（ctx 注入 DB）
- `db`：gorm 初始化 helper
- `redis`：redis 初始化 helper
- `http`：Gin server + gint wrapper + middleware
- `rpc`：gRPC server + interceptor + error detail
- `outbox`：model + repo + publisher 接口

---

## 健康检查

- HTTP：`/health` 或 `/ready`
- gRPC：`grpc.health.v1.Health`

框架默认生成。

---

## 优雅关闭

框架 `App.Run()` 内置：
- 监听 SIGTERM/SIGINT
- 关闭 HTTP/gRPC server
- 关闭 DB/Redis 连接
- 超时强制退出
